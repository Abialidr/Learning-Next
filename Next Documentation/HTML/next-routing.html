What is routing

Basically routing means to open a specific file when navigated in url bar in browser.
Normally browser offers "History api" to do routing.
Next js Also uses HIstory API's internally to do page based routing

Rules for routing

In the NEXTJs, there are two basic rules for creating routing components.

  • The component should be a react component that returns JSX.
  • Instead of basic export, the component should be exported as default.

Index Routing

Any file that is directly added to the NextJS pages folder is referred to as a as index routes.
index routes also point to index.js files as well.
They have no names but are directly related to their parent component.
for example 1, page/index.js is shown as "localhost:3000/". instead of "localhost:3000/index".
for example 2, Page/hello-world/index.js is shown as "localhost:3000/hello-world". instead of "localhost:3000/hello-world/index".

Nested Routing

A "nested route" is any folder that is created within a pages folder.
Inside this folder, index.js is the default route; any other file with a different name is added beside nested-route.
for example, "pages/hello-world/index.js" can be accessed by writing "localhost:3000/hello-world" but "pages/hello-world/hello-world.js" can be accessed by writing "localhost:3000/hello-world/hello-world."

Dynamic Routing

Dynamic routes are pointed toward those routes that have no name but an id
This is a route that can be anything. You add a dynamic id beside a route. to access a definite value dynamically without creating the same page for hundreds of times.
